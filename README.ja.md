## クイックスタート

[Node.js 18.17+](https://nodejs.org/) と、次のいずれかが必要です：API キー、認証済みの [Codex CLI](https://developers.openai.com/codex/cli)、または認証済みの [Claude Code CLI](https://code.claude.com/docs/en/overview)。

```bash
npm install -g penecho
penecho configure
penecho
```

対話的に起動すると、現在のバージョンがすぐに表示されます。サーバーが待ち受けを開始した後、PenEcho は `Checking latest PenEcho version...` と表示し、利用可能になるタイミングを遅らせることなく npm に問い合わせます。新しいバージョンがある場合は、既定値が `Y` のプロンプトで Enter を押すとグローバルにインストールされます。その後、現在のサービスはバックグラウンドプロセスを起動せずに停止します。更新後のバージョンを起動する準備ができたら、もう一度 `penecho` を実行してください。インストール済みのバージョンが最新の場合は、その旨が明示されます。オフラインでの確認と非対話的な起動は、実行中のサービスを妨げません。

`penecho configure` は対話式の設定センターを開きます。メインメニューには `LLM source`、`Settings`、`Exit` があります。矢印キーと Enter で操作します。

- `LLM source -> Claude CLI`：検出済み、推奨、既定、または手動入力したモデルと effort レベルを選択します。Opus 4.8 以降を推奨します。Sonnet と Opus 4.6 も応答できますが、キャンバス上の結果が弱くなる場合があります。
- `LLM source -> Codex CLI`：モデルと effort を選択します。良好な結果には GPT-5.5 以降が必要です。`gpt-5.6-sol` を推奨し、一覧で最も高い Codex effort は `xhigh` です。
- `LLM source -> API`：OpenAI 互換または Anthropic/Claude 互換のリクエスト形式を選び、URL、モデル、effort、非表示のキーを入力します。Kimi K3（[中国](https://platform.kimi.com?aff=penecho) / [グローバル](https://platform.kimi.ai?aff=penecho)）は OpenAI 互換形式とモデル `k3` を使用し、現在のテストでは `medium` を推奨しています。Anthropic API では `none` を選ぶと思考を無効化でき、新しい構成には推奨される `medium` の適応的思考レベルが既定で設定されます。PenEcho は API リクエスト、接続確認、CLI 呼び出しのいずれにも `temperature` を設定しません。各プロバイダーとモデルの既定値が使われます。既存の値は既定値として表示され、キーを空欄にすると保存済みのキーが維持されます。
- `Settings`：モデル共通のタイムアウト、各モデル実行系へ送る画像形式、リクエストの記録と保持、待ち受けインターフェースとポート、Auto AI の初期遅延を設定します。既定は WebP で、PNG も選択できます。遅延はキャンバス上からも変更できます。

各 LLM ページの最後には `Test & Save` があり、PenEcho は確認前に必ず保存します。Codex CLI は高速なオフライン確認を行います。実行ファイルとログインを確認し、`codex debug models --bundled` を読み取って選択モデルが存在することを確かめます。推論の実行、画像の添付、オンラインカタログの更新、モデルのトークン消費は行いません。Claude CLI と API の設定では、選択したエンドポイントとモデル設定を検証するため、小さな実リクエストを一度送信します。確認の成否にかかわらず設定は保存され、UI は明確な診断結果を表示して親メニューへ戻ります。

キャンバスのツールバーには、Auto AI の隣に幅固定でクリック可能な `Reasoning` メニューがあり、リクエストごとに頻繁に変更できます。選択肢は `Configured`、`none`、`low`、`medium`、`high`、およびプロバイダーが実用上対応する最上位レベルです。`Configured` はリクエストごとの effort フィールドを省略し、設定済みのカスタム effort または基盤となる CLI の既定値をサーバーに維持させます。最後の明示的な位置は、OpenAI API と Codex CLI では `xhigh`、Anthropic API と Claude CLI では `max` に対応します。`none` は OpenAI に `reasoning_effort=none` を送信し、Claude では思考を無効化します。モデルの対応状況はプロバイダーに依存するため、選択モデルが実装していないレベルをエンドポイントが拒否する場合があります。保存済みの `AI_EFFORT` がこのコントロールの初期値となり、ツールバーで変更すると、設定ファイルを書き換えずに以降のキャンバスリクエストへ上書き適用されます。メニューは選択後、または操作のない状態が 5 秒続くと閉じます。

既定の設定ファイルは `~/.penecho/config.env` です。API 認証情報はこのローカルファイルに平文で保存され、POSIX システムでは所有者だけがアクセスできる権限が設定されます。ブラウザーのコードへ送信されることはありません。他の認証情報と同様に保護してください。このファイルが存在する前に `penecho` を起動すると、対話式ターミナルで設定センターが自動的に開きます。

必要に応じて、特定の起動に別の env 形式の設定ファイルを使用できます。

```bash
penecho configure --config ./team.env
penecho --config ./team.env
```

明示的に指定した `--config` ファイルは、そのコマンドにおける既定のグローバルファイルを置き換えます。PenEcho は、プロジェクトディレクトリの `.env` やパッケージディレクトリの `.env` を自動的には読み込みません。

<a id="cli-prerequisites"></a>

### CLI の前提条件

Codex デスクトップアプリをインストールしただけでは、シェルの `PATH` から `codex` 実行ファイルを利用できるとは限りません。Codex を選択する前に、CLI を別途インストールして認証してください。

```bash
npm install -g @openai/codex@latest
hash -r
codex --version
codex login status
```

必要であれば `codex login` を実行します。Claude CLI モードでも同様に、インストールして認証済みの Claude Code CLI が必要です。通常は `claude auth login` で認証します。

PenEcho は選択した CLI をローカルで利用するため、この実行元に API キーは不要です。通常の起動では、モデルのトークンを消費せずに実行ファイルとログインを確認します。Codex の `Test & Save` では、モデルリクエストを行わずに、インストール済み CLI の組み込みカタログに選択モデルがあるかも確認します。Claude の `Test & Save` は小さな実リクエストを送信します。

Codex 経由のキャンバスリクエストには `codex exec --json` を使用します。Codex が最終エージェントメッセージと `turn.completed` を出力すると、PenEcho はすぐに応答を返します。その後も CLI プロセスが残っている場合は、キャンバス応答を遅らせる代わりに、バックグラウンドで終了・クリーンアップします。

Claude CLI のリクエストは、ツール、エージェント、MCP、プロンプト候補、セッション永続化、そのほか不要なバックグラウンド通信を無効にした、独立した 1 回の `claude -p` ターンを使用します。effort に `none` を選ぶと `MAX_THINKING_TOKENS=0` が設定され、Claude Code は `thinking.type=disabled` を送信します。`none` は有効な Claude CLI effort 値ではないため、PenEcho は内部的な `low` effort とプロセス単位の `--settings` 上書きも渡し、ユーザーレベルの `CLAUDE_CODE_EFFORT_LEVEL=max` を無効化します。`low`、`medium`、`high`、`max` のいずれかを選ぶと思考は有効なままで、Claude の `--effort` フラグと同じ設定上書きの両方を通じて選択値が適用されます。PenEcho はストリームを逐次検証し、Claude が成功を示す最終 `result` を出力するとすぐに応答します。ツールの使用を試みるとリクエストは中止され、結果後も CLI プロセスが残る場合はバックグラウンドで終了・クリーンアップされます。

一時的な起動時上書きも引き続き利用できます。

```bash
penecho doctor --codex
penecho --codex --model gpt-5.6-sol --effort xhigh
penecho --claude --model opus --effort max
penecho --port 4000
```

`--model`、`--effort`、`--port` はそのプロセスだけに適用され、選択した設定ファイルより優先されます。省略すると、保存済みの選択または基盤となる CLI の既定値を使用します。その他のモデル固有の effort 文字列も受け入れられ、そのまま渡されます。

<a id="run-from-this-source-directory"></a>

### このソースディレクトリから実行する

依存関係をインストールし、インストール済み CLI と同じ本番用エントリポイントを通じて、このチェックアウトを起動します。

```bash
npm install
npm start
```

必要な場合、最初の対話的起動時に設定センターが開きます。`--` より後の引数は PenEcho へ渡されます。例：`npm start -- --port 4000`。

代わりに、このチェックアウトの `penecho` コマンドをグローバルに公開するには、次を使用します。

```bash
npm link
penecho configure
penecho
```

`npm link` はローカルのコマンドリンクを作成しますが、パッケージを公開することはありません。個別のビルド手順はありません。

[http://localhost:3888](http://localhost:3888) を開きます。既定の `0.0.0.0` で待ち受ける場合、起動時には次の行にこのマシンの具体的な LAN URL も表示されます。同じネットワーク上の別端末でそのいずれかを開いてください。接続できない場合は、ホスト OS のファイアウォールまたは適用されるルーティングポリシーで、設定済みの受信 TCP ポートを許可してください。

<a id="think-on-the-canvas"></a>

## キャンバス上で考える

キャンバスの好きな場所に、質問、数式、図、または形になりかけたアイデアを書いて、少し待ちます。PenEcho は書き込みとその空間的な関係を読み取り、隣に回答します。すべての手順をチャットメッセージへ変換したり、融通の利かない作図ツールで作り直したりすることなく、問題に取り組めます。

- 回答、ヒント、説明、続き、数式、プロット、図をキャンバス上で直接受け取れます。
- AI の下書きをキャンバス上で直接ドラッグし、グループ単位または軸ごとにサイズを変え、返されたテキストや数式をコピーできます。作業内容の一部になる前に、採用または破棄します。
- スタイラスやマウスで自然に描き、疎な `20,000 x 20,000` キャンバスをパン・ズームできます。
- 確定済みのインクをフリーハンドの投げ縄で囲み、移動、サイズ変更、色変更、削除、または選択範囲だけを Typeset へ送信できます。通常の選択編集やキャンセルで AI リクエストが発生することはありません。
- 取り組む問題に合わせて Arcane、Sci-fi、Research、Studio モードを選べます。
- 軽量なスナップショットをブラウザー内にローカル保存できます。新しいキャンバスを始める際は、現在のスナップショットを上書きする、新しいコピーを保存する、または保存せず続行することを選べます。未確定の AI 下書きは含まれません。
- 確定済みのキャンバスインクを、各辺に `512` ピクセルの紙面タイル 1 枚分の余白を付けた、切り抜き済み PNG として書き出せます。

PenEcho は小さなローカルランタイムを維持し、インクのある場所にのみ `512 x 512` タイルを割り当てます。そのため、巨大な論理キャンバスが巨大なビットマップになることはありません。

<a id="whats-new-in-070"></a>

## 0.7.0 の新機能

- **キャンバス上のライブでインタラクティブな HTML。** 新しい General HTML プラグインにより、モデルは時計、計算機、ダッシュボードなど目的を絞ったレスポンシブなインターフェースを、サンドボックス化されたキャンバスウィジェットとして構築できます。ウィジェットの内容は直接操作でき、長押しするとキャンバス編集へ切り替わり、移動、サイズ変更、確定、削除ができます。
- **PenEcho のデータサービスなしで役立つデータを取得。** 専用プラグインは、天気、株価、技術ニュース、為替レート、地震、自然現象、宇宙天気、GitHub アクティビティを対象とします。リクエストはユーザーのブラウザーから各プラグインが宣言した API オリジンへ直接送られます。PenEcho がプラグインデータをプロキシまたはキャッシュすることはありません。
- **明示的なセキュリティ境界。** 各データプラグインは許可する `connect` オリジンを宣言します。ウィジェットのネットワーク通信はその許可リストに限定され、HTML は独立した iframe で実行されます。無効化したプラグインは、プロンプト、ペイロード、メッセージフック、ウィジェットのランタイム動作のいずれにも影響しません。
- **ローカルでのプラグイン作成。** コンパクトな Markdown 形式に、メタデータ、ランタイム規則、必須のワンショット例を、およそ 1,000 トークン以下のプロンプト予算でまとめます。Preview 作成ツールは AI で下書きを改善し、タイトルを補完し、ローカルに保存して有効化できます。個人用プラグインは後から削除でき、カタログでは個人用と組み込みプラグインが明確に分けられます。
- **キャンバスネイティブの永続化と書き出し。** 確定済みウィジェットはローカルスナップショットと PNG 書き出しに含まれます。幅と高さのハンドルはテキストを拡大縮小せずにインターフェースを再配置し、角のハンドルは画像と同じようにウィジェット全体を拡大縮小します。削除は引き続き元に戻せます。
- **妥当な既定値。** 新規ユーザーには General HTML、Animation scenes、Weather が既定で有効になります。それ以外のデータプラグインはオプトインのままで、選択内容はすべてローカルに記憶されます。

<a id="whats-new-in-060"></a>

## 0.6.0 の新機能

- **制御可能なアニメーション説明。** 宣言型アニメーションシーンが既定で有効になり、明示的に求めた場合、または動きによって説明が実質的に分かりやすくなる場合にモデルがアニメーションを使用できます。一般の Plugins メニューから無効化すると、追加の 500〜600 プロンプトトークンを取り除き、元のリクエストと描画動作へ戻せます。
- **安全で永続的なアニメーション描画。** 透明な Canvas2D シーンは最大 32 個のオブジェクトと 32 個のモーションをサポートし、すぐに再生され、確定の前後を問わず編集でき、スナップショットに保存されます。レンダラーはダーティ領域更新を行う独立した透明レイヤーを使用し、キャンバスを 20 個のアニメーションに制限します。モデルが提供する JavaScript を実行することはありません。
- **タッチとペンに適したアニメーション操作。** マウスまたはペンでクリックすると、アニメーションツールバー全体がすぐに表示されます。タッチでは、キャンバスのパン中に誤作動しないよう 1 秒間の長押しが必要です。ツールバーと選択範囲の輪郭は、10 秒後、外側のクリック、または再度描画を始めると同時に非表示になります。
- **より信頼性の高いモデル出力。** リクエストは出力用の余裕を確保し、過大なシーンを促さずにアニメーション寸法を制限します。また、サーバーは無害な末尾出力から最初の完全な JSON レスポンスを復元できます。無効、不完全、無効化済み、または上限を超えるアニメーションコマンドは明示的に拒否されます。
- **より鮮明なテキストと下書き描画。** 確定したテキストは安全な Markdown と LaTeX らしい記述を自動整形し、Preview でも同じ最終結果を表示します。大きくダウンサンプリングされた AI 下書きは論理寸法に合わせてクリップされ、最初に左上の 4 分の 1 だけが描画される不具合を防ぎます。
- **処理を妨げない npm 更新。** 対話的な起動は npm を確認する前にサービスを利用可能にし、インストール済みバージョンを分かりやすく表示します。グローバル更新を受け入れると正常に停止するため、更新後のサービスを手動で起動できます。

<a id="animation-scenes-in-060"></a>

## 0.6.0 のアニメーションシーン

アニメーションは取り外し可能なプラグインであり、すべての PenEcho リクエストに常に必要なものではありません。キャンバスツールバーの `Plugins` メニューでは `Animation scenes` が既定で有効です。元の静的リクエストとキャンバス動作を使いたい場合は、いつでもチェックを外してください。PenEcho はその選択を記憶します。

- **有効時：** 明示的にアニメーションを求めた場合、または動きが回答の理解を実質的に助ける場合、モデルはアニメーションデモを返せます。追加のアニメーションプロトコルにより、AI リクエストごとに約 `500–600` 入力トークンが加わります。
- **無効時：** PenEcho はアニメーションプロトコルとプラグインフィールドを省き、新しいアニメーションコマンドを除外し、アニメーションシーンを描画しません。静的リクエストとキャンバス動作は、この機能の導入前と同じです。以前に保存したシーンは保持され、プラグインを再び有効にすると利用できます。
- **透明であることを前提に設計：** アニメーションシーンには背景レイヤーがないため、下にある紙面、手書き、図、そのほかのキャンバス内容が見えたままになります。

たとえばプラグインを有効にすると、軌道モデルのアニメーション、動く幾何学的作図、相互作用する図形、または時間変化が説明の一部となる別の過程を記述・依頼できます。モデルには実際の依頼に合うサイズを選ぶよう指示されます。各軸 `5000` 論理単位は上限であり、目標値ではありません。

返されたシーンは編集可能な下書きの段階から再生を開始します。下書きを移動またはサイズ変更し、その後、ほかの AI 出力と同じように採用または破棄します。確定後は次の操作ができます。

- マウスまたはペンでクリックすると、完全な編集コントロールがすぐに表示されます。
- タッチで 1 秒間長押しすると、通常の 1 本指によるキャンバスのパンを妨げずに表示できます。
- コントロールを使って、一時停止または再開、最初から再生、移動、サイズ変更、編集の確定、編集のキャンセル、シーンの削除を行えます。
- 約 10 秒後、外側のクリック、キャンバスの移動、ツール変更、または新しいペンストロークにより、ツールバー、輪郭、サイズ変更コントロールが同時に非表示になります。

アニメーションシーンは、PenEcho 独自の Canvas2D レンダラーで描画される宣言型 JSON データです。モデルが提供する JavaScript は実行されません。各シーンは `32` 個のオブジェクトと `32` 個のモーションに制限され、1 つのキャンバスには最大 `20` 個のアニメーションを置けます。表示中かつ再生中のシーンだけがアニメーションフレームを要求します。複雑な表示シーンは更新頻度を下げ、変化した画面領域は個別に再描画されます。確定済みのシーンと再生状態はローカルスナップショットへ保存され、有効なアニメーションはプレビュー、書き出し、以後のモデルコンテキストで固定フレームとして取り込まれます。

## 推奨モデル構成

以下の推奨構成は、PenEcho の実際のキャンバス処理における回答品質と待ち時間のバランスを取ったものです。合成ベンチマークではなく、現在の実機テストに基づいています。実際の応答時間は、プロバイダー、キャンバスの複雑さ、画像サイズ、推論動作によって変化します。

| モデル | Effort | 品質と速度 | 推奨用途 |
| --- | --- | --- | --- |
| `claude-opus-4-8` | `medium` | 高品質で、待ち時間とのバランスが良い | 日常的なキャンバス作業に推奨する Opus の既定値 |
| `claude-opus-4-8` | `high` | 推論品質はより高いが、待ち時間が長く変動も大きい | 手書き、数学、図、レイアウト判断など、速度より品質が重要な複雑な作業 |
| Fable 5（`claude-fable-5` または `fable`） | `medium` | 非常に良い結果。現在のテストでは、`xhigh` の `gpt-5.6-sol` と比べて応答時間が半分程度になることが多い | 高速かつ高品質な汎用候補 |
| [Kimi K3](https://platform.kimi.ai?aff=penecho)（`k3`） | `medium` | 現在の比較で非常に高品質。`medium` により実用的な品質と速度のバランスを保てる | 負荷の高いキャンバス作業に推奨する Kimi/API の既定値 |
| `gpt-5.6-terra` | `low` から `high` | 驚くほど高性能で応答も速い。現在の PenEcho キャンバステストでは、高速に応答しながら `gpt-5.6-sol` より良い結果を生成 | 品質と待ち時間の目標を柔軟に調整できる推奨 OpenAI オプション |
| `gpt-5.6-luna` | `xhigh` | キャンバス結果が非常に良く、応答も速い | `xhigh` 推論が適切な場合の、応答性に優れた品質優先オプション |
| `gpt-5.6-sol` | `high` | 多くのリクエストに十分で、`xhigh` より応答が速い | 応答性を重視する場合に推奨する Sol の既定値 |
| `gpt-5.6-sol` | `xhigh` | 非常に良い結果だが、低速で変動も大きい | 難しいキャンバス作業向けの品質優先 Sol 構成 |

Google モデルはまだテストされていません。貢献を歓迎します。Gemini または別の Google モデルを試した場合は、モデル ID、プロバイダーまたは実行系、推論構成、おおよその待ち時間、代表的なキャンバス例を Issue で共有してください。

<a id="token-use-and-cost"></a>

## トークン使用量とコスト

一般的な PenEcho のキャンバスリクエストでは、プロバイダーが報告する場合の非表示の推論トークンを含む総出力使用量は、選択した effort レベルに応じておおよそ次のようになります。

| Effort | 一般的な出力使用量 | 実用上の目安 |
| --- | --- | --- |
| `low` | 約 `1,000` トークン | 多くの日常的なキャンバスリクエストには通常十分 |
| `medium` | 約 `3,000` トークン | 認識、数学、図、レイアウトにより多くの推論余地を確保 |
| `xhigh` または `max` | 約 `5,000–8,000` トークン | 品質優先の `gpt-5.6-sol` やその他の最大 effort リクエストで一般的。待ち時間とコストの増加を見込む必要がある |

これは実用上の推定であり、PenEcho が強制する上限ではありません。実際の使用量は、モデル、プロバイダー、キャンバスの複雑さ、推論動作によって大きく変わる場合があります。以下のコスト例では、一般的な `low` のケースである `10,000` 入力トークンと `1,000` 出力トークンを引き続き使用します。標準的な短いコンテキストの API 料金では次のようになります。

- `gpt-5.6-sol`：`10,000 x $5.00 / 1M + 1,000 x $30.00 / 1M = $0.080`
- `gpt-5.6-terra`：`10,000 x $2.50 / 1M + 1,000 x $15.00 / 1M = $0.040`
- `gpt-5.6-luna`：`10,000 x $1.00 / 1M + 1,000 x $6.00 / 1M = $0.016`

これらの例では、1 リクエストあたり約 1.6〜8 セントです。Medium、`xhigh`、`max` のリクエストでは推論トークンが出力として課金されるため、より高額になる場合があります。価格は変更されることがあるため、現在の料金は [OpenAI API の料金](https://developers.openai.com/api/docs/pricing)ページで確認してください。

ChatGPT で Codex にサインインした場合、PenEcho は API キーではなく、プランに含まれる Codex 使用枠を使います。利用枠はプランによって異なり、追加利用には ChatGPT クレジットが必要になる場合があります。現在のプランと制限については [Codex の料金](https://learn.chatgpt.com/docs/pricing)を参照してください。同様に Claude CLI モードは Claude Code で認証済みのアカウントを使用し、Anthropic API の課金とは別です。

<a id="safe-deployment"></a>

## 安全なデプロイ

PenEcho は既定で `0.0.0.0:3888` を待ち受けるため、localhost と信頼できる LAN からすぐにアクセスできます。実行系に合うデプロイ境界を選んでください。

- **Codex CLI および Claude CLI モード：** ローカルマシンまたは信頼できる直接接続の LAN だけで使用してください。有効なリクエストはローカル CLI プロセスを起動するため、どちらのモードも公開インターネットや信頼できないリバースプロキシへ直接公開しないでください。どちらも公開オリジンの設定なしで localhost と LAN アドレスからすぐに利用できます。PenEcho は、選択した CLI を起動する前に、Host、クライアントネットワーク、完全一致する Origin、プロセス存続期間中のセッション Cookie、JSON コンテンツタイプを確認します。新しい有効なリクエストは、その直前のリクエストをすぐに置き換えます。キューで待機したり、ビジー応答を返したりすることはありません。
- **API モード：** ローカル、LAN、プロキシ、リモートのリクエストを、PenEcho レベルの Host または Origin 制限なしで意図的に受け入れます。公開する場合は、HTTPS、認証、レート制限、リクエストサイズ制限の背後に配置してください。選択した設定ファイルとプロバイダーキーを非公開に保ってください。認証情報は Node.js プロセス内に留まり、ブラウザーコードへ送信されることはありません。

どちらのモードでも、問題を積極的に診断している場合を除き、本番環境ではデバッグ成果物とリクエストトレースを無効にしてください。また、非公開の内容を含む設定ファイル、ログ、スクリーンショット、保存済みリクエストを公開しないでください。`Settings` でリクエスト記録を有効にすると、有効な AI リクエストは既定で `~/.penecho/logs/requests` 以下へ保存されます。これには元の `atlas.png`、送信画像、認証情報を除去したリクエスト本文、未加工および解析済みの応答、フォールバックの詳細、最終状態が含まれます。UI にはこのパスも表示され、保持数を設定できます。

<a id="useful-configuration"></a>

## 便利な設定

設定センターは、以下の設定を `~/.penecho/config.env` または `--config` で選んだファイルへ書き込みます。

| 設定 | 用途 |
| --- | --- |
| `AI_PROVIDER` | 実行系：`api`、`codex-cli`、または `claude-cli` |
| `AI_API_FORMAT` | API リクエスト形式：`openai`（既定の例）または `anthropic` |
| `AI_API_URL` / `AI_API_KEY` | API エンドポイントと認証情報。API モードだけで使用 |
| `AI_API_MODEL` | API モードで使用するモデル |
| `AI_EFFORT` | キャンバスコントロール起動時の推論 effort。ツールバーの `Configured` はカスタム名を含めて値をそのまま維持し、空の CLI 値では CLI の既定値が保持される。Anthropic API と Claude CLI は `none`（送信値は `thinking.type=disabled`）に対応し、OpenAI は `reasoning_effort=none` を送信する。ツールバーで明示的に選んだレベルは、設定ファイルを変更せずにリクエストごとに上書きする |
| `AI_TIMEOUT_SECONDS` | API、Codex CLI、Claude CLI のモデル試行に共通するタイムアウト。既定は 180、指定範囲は 10〜600 |
| `PENECHO_AI_IMAGE_FORMAT` | API、Codex CLI、Claude CLI へ送る画像形式：`webp`（既定）または `png` |
| `CODEX_CLI_MODEL` | Codex CLI モードの任意のモデル上書き |
| `CLAUDE_CLI_MODEL` | Claude CLI モードの任意の別名またはモデル ID 上書き |
| `AUTO_AI_DELAY_SECONDS` | 自動認識までの初期遅延。ブラウザーコントロールから 0〜10 秒の範囲で上書き可能 |
| `PENECHO_REQUEST_TRACE` | ローカルでリクエストごとの画像、送信リクエスト、応答、結果のトレースを保存。既定では無効 |
| `PENECHO_REQUEST_TRACE_LIMIT` | 保持するローカルリクエストトレースの数。既定は 100、最大は 1000 |
| `HOST` / `PORT` | 待ち受けインターフェースとポート。既定は `0.0.0.0:3888` |

インストール済み CLI の起動では、`--model` が選択実行系のモデル設定を上書きし、`--effort` がそのプロセスに限って `AI_EFFORT` を上書きします。コマンドラインオプションとプロセス環境変数は、選択した設定ファイルより優先されます。

変更を提出する前に、次の確認を実行してください。

```bash
npm run check
```

実装の詳細は[アーキテクチャノート](docs/architecture.md)を参照してください。

## 一緒に開発する

PenEcho はまだ若く、オープンに開発されているプロジェクトです。手書き認識、キャンバス上のビジュアルツール、より幅広いモデル対応、自然なペン操作など、重要な課題はまだ残っています。開発を前進させるために、コードを書く必要はありません。

参加方法：

- **使って、結果を共有する。** うまく動いたキャンバスも、うまくいかなかったキャンバスも、どのベンチマークより多くの情報を与えてくれます。機密情報を取り除いたスクリーンショットだけでも十分です。
- **モデルをテストする。** PenEcho では API、Codex CLI、Claude CLI の実行方式ごとに独立してモデルを選択でき、動作はモデルごとに異なります。実行系、モデル ID、effort、おおよその待ち時間、結果の例を報告してください。Google/Gemini モデルは未検証のため、特に歓迎します。
- **使いにくい点を報告する。** 認識ミス、レイアウト不具合、不自然なペン操作は、どれほど小さくても Issue にする価値があります。モデル固有の問題では、再現可能なキャンバス例と期待結果・実際の結果を含めてください。
- **コードを書く。** 認識、ビジュアルツール、モデルアダプター、ペン入力には、いずれも改善の余地があります。Pull Request を開く前に `npm run check` で完全なチェック一式を実行してください。
- **より多くの人が読めるようにする。** UI は現在、英語と中国語を提供しています。翻訳の追加や、より明確なドキュメントを歓迎します。

交流先：

- [Discord](https://discord.gg/3jrPJ3mXdX) — リアルタイムの議論、モデルテストの記録、共有キャンバスのワークフロー。新しく参加する方や作業中のものも歓迎します。
- [GitHub Discussions](https://github.com/penecho/penecho/discussions) — 検索可能な形で残しておきたいアイデアや質問。
- [GitHub Issues](https://github.com/penecho/penecho/issues) — 再現可能な不具合と確認済みの作業。

初めて貢献する方は [CONTRIBUTING.md](CONTRIBUTING.md) から始めてください。PenEcho が気に入ったら、リポジトリに Star を付けてデモを共有してください。その可視性が次の参加者につながります。
